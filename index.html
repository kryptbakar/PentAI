About

PentAI turns a manual penetration-testing pipeline — subdomain enumeration, service probing, fingerprinting, CVE scanning, and triage — into a single action: type a domain. Every scan passes through a hard authorization chokepoint that checks an allow-list, signed rules-of-engagement, and an active engagement window before any tooling runs, with every allowed and denied attempt written to an append-only audit log. Real, industry-standard tools (nmap, subfinder, httpx, nuclei) execute in throwaway containers, and when configured, Claude turns raw findings into business-risk-ranked, deduplicated, remediation-backed reports surfaced in a glassmorphic security operations console.

Highlights

  • One-click assessmentsCommand Center provisions scope and dispatches the full recon → vulnerability toolchain from a single domain input.
  • Authorization-firstEvery scan passes a single chokepoint enforcing allow-list, RoE, and time-window checks before execution.
  • Real toolingnmap, subfinder, httpx, and nuclei run in isolated, throwaway containers — no mock output.
  • AI-assisted triageClaude re-ranks findings by business risk, deduplicates across tools, and drafts remediation guidance.
  • SSRF / internal-target guardHard refusal of loopback, RFC1918, link-local, and cloud-metadata targets, unit-tested in CI.
  • Premium consoleLive scan status, severity analytics, CVE-referenced findings, and exportable engagement reports.
  • Standalone CLIThe pentai terminal tool runs DNS, header, and TLS checks with no database or server required.
  • Continuous monitoringScheduled re-scans, findings diffing over time, and outbound alerting on new critical findings.
